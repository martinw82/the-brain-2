# SESSION-BASED BRANCHING WORKFLOW ## MANDATORY STARTUP SEQUENCE (DO IN ORDER): 1. Run: git status 2. Read: agent-brief.md (project context) 3. Read: brain-status.md (current state - check "Next Task") 4. Read: brain-roadmap.md (find FIRST unchecked item) 5. Read: dev-log.md (note: currently out of sync - update it!) 6. Run: git log --oneline -3 (see recent commits) 7. IF uncommitted changes: STOP and ask user 8. Create branch: git checkout -b session-XXX-task-name ## DUTIES: - Work on FIRST unchecked task in brain-roadmap.md - Update brain-status.md after EVERY sub-task - Update dev-log.md (bring back in sync!) - Commit: git commit -m "type: description [Session XXX]" - Check off completed items in brain-roadmap.md (- [x]) IF NO TASKS LEFT CHECK WITH USER AND DISCUSS NEXT TASKS ## SHUTDOWN: 1. Final commit 2. Run tests 3. IF tests pass: merge to main, push, delete branch, update dev-log.md 4. IF tests fail: keep branch, update status.md with blocker
